# 게임 디자인 분석 노트 — 재사용 자료집

> 용병단연대기 개발 과정에서 정리한 장르 분석 + 설계 공식 모음.
> **다음 게임을 만들 때 이 문서를 먼저 읽고 시작할 것.**
> 마지막 갱신: 2026-07 (용병단연대기 v2.0 개발 시점)

---

## 0. 이 문서 사용법

새 게임을 구상할 때 순서:

1. **§2 공통 설계 패턴**부터 읽는다 — 장르 불문 적용되는 뼈대.
2. 만들려는 장르와 가까운 게임을 **§1**에서 찾아 "훔칠 아이디어 / 함정" 확인.
3. **§3 밸런스 공식**으로 초기 수치를 잡는다 (감으로 잡지 말 것).
4. 출시 직전 **§4 체크리스트**를 전부 통과시킨다.

## 1. 장르별 게임 분석

각 게임을 5개 항목으로 해부: **핵심 루프 / 왜 재밌나 / 리텐션 장치 / 훔칠 아이디어 / 함정(따라하면 안 되는 것)**

### 1-1. Battle Brothers (용병단 경영의 원형)

- **핵심 루프**: 계약 수주 → 턴제 전투 → 보수/전리품 → 부대 유지비 지불 → 더 어려운 계약. 매일 급여·식량이 빠져나가는 **경제 압박**이 모든 결정의 배경.
- **왜 재밌나**: 용병 하나하나가 이름·배경·특성·부상 이력을 가진 "사람"이라 정이 들고, 퍼머데스라서 그 정이 긴장으로 바뀐다. 스토리를 만들어주는 게 아니라 **플레이어가 스토리를 겪게** 한다 (창발적 서사).
- **리텐션 장치**: 전멸 전제의 반복 플레이. 오리진(창단 배경) 선택으로 매 회차 시작이 다름. 야망(ambition) 시스템이 중기 목표를 계속 제시.
- **훔칠 아이디어**: ① 일일 유지비 경제 ② 부상/흉터의 영구 흔적 ③ 계약 협상(선금 vs 성공보수) ④ 야망 = 자동 생성되는 중기 목표.
- **함정**: 가이드 부재로 진입장벽이 악명 높음. 초반 3시간을 넘기지 못하는 유저가 대부분 — **하드코어함과 불친절함은 다르다.**

### 1-2. Wartales (접근성으로 성공한 후발주자)

- **핵심 루프**: Battle Brothers와 동일 소재이나, 고정 맵 + 스토리 라인 + 명확한 직업 스킬트리로 학습장벽을 낮춤.
- **왜 재밌나**: "무엇을 해야 할지"가 항상 보인다. 지역마다 서사가 있고, 캠프(야영) 운영이 부대에 생활감을 부여.
- **리텐션 장치**: 지역 단위 콘텐츠 개방, 직업 조합 실험, 지속적 DLC.
- **훔칠 아이디어**: ① 하드코어 장르 + 캐주얼 온보딩 조합 ② 캠프/야영 = 전투 밖에서 부대에 정드는 시간 ③ 고정 맵의 지역별 위기.
- **함정**: 중반 이후 반복 작업(크래프팅 노가다)이 늘어짐. 접근성을 높이면 **깊이로 보상**해야 후반이 산다.

### 1-3. Darkest Dungeon (연출과 포지셔닝의 교과서)

- **핵심 루프**: 원정 준비 → 4인 포지션 전투 → 스트레스/기벽 누적 → 마을에서 회복·치료 → 재정비.
- **왜 재밌나**: ① 전열/후열 4칸 포지셔닝 — 같은 캐릭터도 서는 위치에 따라 쓸 수 있는 스킬이 달라 배치 자체가 퍼즐 ② 스트레스라는 두 번째 체력 — HP는 멀쩡한데 정신이 무너지는 이중 긴장 ③ 내레이터의 목소리와 판화풍 아트가 만드는 압도적 분위기.
- **리텐션 장치**: 영웅은 소모품이라는 설계(죽으면 새 영웅 영입) + 마을 시설 영구 업그레이드(= 메타 진행).
- **훔칠 아이디어**: ① 포지션이 전술의 핵심 축 ② "두 번째 자원" (스트레스/사기/광기 등) ③ 마을 시설 = 회차 간 유지되는 성장 ④ 톤앤매너의 일관성 (색·서체·문장까지 하나의 목소리).
- **함정**: 후반 그라인드가 심함. RNG 불운이 "내 실수"가 아니라 "게임 탓"으로 느껴지는 순간 이탈 — **불운도 예고(적 의도 표시 등)로 완화해야 한다.**

### 1-4. Slay the Spire (반복 플레이 설계의 표준)

- **핵심 루프(런 구조)**: 시작 → 분기 맵 선택 → 전투/이벤트/상점 → **3택1 카드 보상** → 보스 → 사망 or 클리어 → 처음부터. 한 런 30분~1시간.
- **왜 재밌나**: 매 런이 다른 빌드가 된다. "이번엔 독 덱이 떴다"라는 **덱 정체성의 발견**이 재미의 본체. 죽어도 카드 지식이 남아서 손해가 아님.
- **리텐션 장치**: 승천(Ascension) 20단계 난이도, 일일 도전, 언락 카드. "한 판만 더"의 왕.
- **훔칠 아이디어**: ① **3택1 보상** — 선택지가 3개일 때 고민이 가장 즐겁다 (2개는 심심, 4개+는 피로) ② 유물(relic) = 런 한정 패시브로 매 런 색깔을 바꿈 ③ 적 의도(intent) 표시 — 다음 행동을 보여주면 운빨이 전술이 된다 ④ 시드 기반 런.
- **함정**: 콘텐츠 풀이 작으면 3택1이 금방 "또 이거네"가 됨. **선택지 시스템은 풀 크기가 생명.**

### 1-5. FTL / Into the Breach (작은 스코프, 완전한 게임)

- **핵심 루프**: FTL — 노드 이동 → 조우 → 자원 관리 → 최종 보스. Into the Breach — 8×8 그리드에서 **완전 정보** 퍼즐 전투.
- **왜 재밌나**: Into the Breach는 적의 다음 행동이 100% 공개된다 — 운이 아니라 순수하게 "내 수읽기"의 게임. FTL은 자원(연료·선원·선체)이 전부 연결된 긴축 경제.
- **훔칠 아이디어**: ① 완전 정보 공개는 턴제의 격을 올린다 ② 스코프가 작아도 시스템이 맞물리면 완전한 게임 ③ 실패해도 짧아서(30분) 다시 시작이 아깝지 않음.
- **함정**: 완전 정보는 밸런스 요구치가 극도로 높다. 수치가 어긋나면 "정답이 없는 퍼즐"이 된다.

### 1-6. 카이로소프트류 (모바일 경영 시뮬의 문법)

- **핵심 루프**: 시설 배치 → 자동으로 돌아가는 경영 관찰 → 수익으로 확장 → 랭킹/이벤트.
- **왜 재밌나**: 숫자가 오르는 것을 "구경"하는 재미 + 배치 퍼즐. 조작 부담이 낮아 자기 전에 하는 게임.
- **리텐션 장치**: 도감·수집(직원, 시설, 장르 조합), 연 단위 이벤트 캘린더, 명확한 단기 목표의 연쇄.
- **훔칠 아이디어**: ① 탭 기반 허브 UI (모바일 문법) ② 도감 = 완성 욕구 ③ "다음에 뭘 하면 되는지"가 항상 화면에 있음.
- **함정**: 후반에 할 일이 사라지면 급격히 식음. 엔드게임 콘텐츠(랭킹, 2회차)를 미리 설계할 것.

### 1-7. 방치형/키우기 (Idle — 쿠키커터가 아니라 심리학)

- **핵심 루프**: 클릭/전투로 재화 → 업그레이드 → **접속 안 해도 쌓임** → 돌아오면 한 번에 수확 → 환생(프레스티지)으로 배수 획득.
- **왜 재밌나**: 손해 보지 않는 게임. 언제 들어와도 보상이 기다림. 환생 시스템은 "지금까지의 성장을 화폐로 바꿔 더 빠른 2회차"라는 천재적 루프.
- **리텐션 장치**: 오프라인 보상, 일일 퀘스트, 프레스티지 배수.
- **훔칠 아이디어**: ① **프레스티지 = 회차 화폐** (용병단연대기의 '유산'이 이것) ② 복귀 보상 ③ 지수 성장 곡선과 로그 스케일 표기.
- **함정**: 서버 없는 웹게임에서 "오프라인 보상"은 localStorage 시간 비교로 구현 가능하지만, 시계 조작에 취약. 경쟁 요소가 없다면 그냥 허용하는 게 낫다.

### 1-8. Vampire Survivors (한 가지 감각에 올인)

- **핵심 루프**: 30분 생존 → 레벨업마다 3~4택1 무기 선택 → 화면을 뒤덮는 파워 판타지 → 사망 → 영구 업그레이드 구매.
- **훔칠 아이디어**: ① 조작은 이동뿐 — **뺄셈의 설계** ② 런 내 성장(무기 진화)과 런 밖 성장(골드 업그레이드)의 이중 구조 ③ 언락이 곧 콘텐츠(캐릭터/무기/맵 해금).
- **함정**: 감각적 쾌감(이펙트·사운드)이 본체라서, 이걸 웹에서 따라하려면 성능 최적화가 관건.

### 1-9. Cookie Clicker / 웹 캐주얼 (브라우저 게임의 생존 법칙)

- **교훈**: 웹게임은 설치 장벽이 없는 대신 **이탈 장벽도 없다.** 첫 15초 안에 "아, 이거구나"가 나와야 하고, 저장은 무조건 자동이어야 하며, 새로고침이 곧 종료 버튼이다.
- **훔칠 아이디어**: ① 단일 파일/즉시 로드 ② 자동 저장 + "이어서 하기" ③ 소리는 기본 켬이되 첫 상호작용 후 재생 (브라우저 자동재생 정책).

## 2. 공통 설계 패턴 (장르 불문)

### 2-1. 코어 루프는 3개 시간 층위로 설계한다

| 층위 | 주기 | 질문 | 용병단연대기의 예 |
|---|---|---|---|
| 마이크로 | 초 단위 | 이번 턴에 뭘 하지? | 공격/스킬/물약/대기 + 목표 선택 |
| 미드 | 분 단위 | 이번 판(하루)을 어떻게 이기지? | 어떤 의뢰, 누구 출전, 전열/후열 |
| 매크로 | 시간 단위 | 이 캠페인의 목표는? | 명성 80 → 최종전 → 유산 → 다음 회차 |

**세 층위 중 하나라도 "고민이 없는" 층이 있으면 그 시간대에 지루함이 발생한다.**

### 2-2. 리텐션의 4대 장치 (하나 이상 필수, 둘 이상 권장)

1. **메타 진행** — 실패해도 남는 것 (유산, 언락, 도감, 업적)
2. **가변 보상** — 매번 다른 것 (유물 3택1, 랜덤 이벤트, 후보 갱신)
3. **미완성 목표** — 항상 보이는 다음 할 일 (업적 12개 중 8개… 나머지 4개는?)
4. **난이도 사다리** — 클리어 후의 산 (도전 규칙, 승천, NG+)

### 2-3. 보상 설계 공식

- **3택1 법칙**: 선택 보상은 3개가 최적. 전부 "좋은 것"이어야 하고, 서로 다른 축(공격/생존/경제)이어야 고민이 생긴다.
- **확정 + 가변**: 보상 = 확정치(보수 골드) + 가변치(30% 확률 물약, 35~50% 유물). 확정만 있으면 지루하고 가변만 있으면 불안하다.
- **손실 프레이밍 주의**: "실패 시 -3 명성"은 "성공 시 +N"보다 2배 아프게 느껴진다 (손실 회피). 패널티는 보상의 절반 이하 크기로.

### 2-4. 온보딩 3분 규칙

- 0~15초: 게임의 정체가 보인다 (타이틀 한 줄 + 시작 버튼)
- 0~3분: 코어 루프를 1회 완주한다 (첫 전투 승리까지)
- 첫 판 안에: 죽어도 되는 안전한 실패를 한 번 겪게 한다
- 튜토리얼은 모달 1장 + 상황별 힌트. **읽게 하지 말고 하게 하라.**

### 2-5. 소프트락 방지 (싱글 게임의 의무)

- 모든 자원이 0이 되는 조합을 나열하고 각각에 탈출구를 심는다.
  - 예: 부대 전멸 + 파산 → 촌장 지원금 이벤트 / 부상 전원 → 휴식으로 회복 / 최약체로 강한 의뢰 → 후퇴 버튼.
- "다시 시작해야만 하는 상태"가 발견되면 그건 버그다.

### 2-6. 웹/단일 파일 게임의 기술 원칙

- 의존성 0 (CDN도 피한다 — 오프라인/차단 환경 대응)
- localStorage 저장은 **버전 필드 + 마이그레이션** 필수 (`{v:2, ...}`, v1이면 필드 보충)
- 저장 키 분리: 회차 저장(run)과 영구 저장(meta)을 다른 키로 — "새 게임"이 업적을 지우면 안 된다
- 인라인 onclick에 문자열 넣을 때 따옴표 충돌 주의 (v1.0에서 실제 버그: `JSON.stringify`가 만든 `"`가 HTML 속성을 깨뜨림 → 항상 `'${id}'` 형태로)
- 한국어 UI: `word-break: keep-all` (단어 중간 줄바꿈 방지)
- 오디오: WebAudio는 사용자 제스처 후에만 시작. BGM/SFX 토글 분리, 설정은 meta에 저장
- 이모지 아이콘: 어두운 배경에서 안 보이는 글리프(♞ 등) 회피, VS16(️) 유무에 따라 흑백 렌더링될 수 있음 — 실제 스크린샷으로 확인

### 2-7. 자동 플레이 테스트 (이번에 확립한 방법론)

- Playwright로 **창단→전투→성장→엔딩→2회차→세이브/로드→패배 경로**를 전부 자동 주파
- `window.__DEBUG` 훅(상태 읽기, 골드/명성 치트)을 게임에 심어두면 테스트가 10배 쉬워짐
- 반드시 **실제 버튼 클릭 경로**로 최소 1회 검증 (JS 직접 호출만 하면 onclick 깨진 버그를 놓친다 — v1.0에서 실증됨)
- 콘솔 에러 0건을 통과 기준으로. 모달이 겹치는 시나리오(결과 화면 위 스토리 모달)를 테스트에 포함

## 3. 숫자 밸런스 공식 모음

실전에서 쓴 초기값. 새 게임에서도 이 비율로 시작한 뒤 플레이테스트로 조정.

### 3-1. 전투 수치

- **TTK(처치까지 턴 수)**: 일반 잡몹 = 아군 1인 기준 2~3방. 보스 = 파티 전체 딜의 4~6라운드.
- **피해 공식**: `dmg = atk × (0.85~1.15 랜덤) × 배율 − def × 0.6`, 최소 1 보장. 방어 계수를 1.0으로 두면 후반 탱커가 무적이 된다 — 0.5~0.7이 안전.
- **치명타**: 기본 5%, 특화 클래스 +10%, 배수 1.5×. 2× 이상은 분산이 너무 커져 턴제에서 불쾌.
- **스킬 배율**: 단일기 1.4~1.8× (쿨 3), 광역기 0.9× (쿨 4), 힐 최대체력의 40% (쿨 2). 광역은 "단일 × 적 수"보다 약해야 함.
- **상태이상**: 도트 피해 = 시전자 공격력의 35% × 2턴. 기절은 1턴 + 보스 면역 필수 (아니면 퍼머스턴으로 보스전 붕괴).

### 3-2. 경제

- **일일 지출 : 최저 의뢰 보수 = 1 : 3~4** (초반 기준). 딱 맞으면 숨막히고, 10배면 긴장이 없다.
- **성장 곡선**: 능력치는 선형(레벨당 +고정), 비용은 완만한 지수(단계당 ×2.2~2.5), 경험치 요구량은 `30 + lvl×25` 같은 선형+상수.
- **장비 티어**: 3티어면 충분. 가격은 110 → 290 → 680처럼 ×2.5, 효과는 +2 → +4 → +7처럼 체감 유지.
- **되팔기 반값**: 장비 교체 시 기존 장비 50% 환불 — 실험을 장려하면서 경제는 지킨다.

### 3-3. 캠페인 페이스

- 목표 캠페인 길이 30~60분(웹게임 기준) = 전투 12~18회 + 휴식/이벤트.
- 명성(진행 게이지) 마일스톤: 0 / 10 / 25 / 50 / 80 — 앞은 촘촘하게, 뒤는 넓게 (초반 도파민 밀도 높임).
- 랜덤 이벤트 확률: 하루당 30~35%. 매일 나오면 소음, 10%면 존재감 없음.

### 3-4. 메타 진행(유산/프레스티지)

- 1회차 클리어 보상으로 영구 업그레이드 1~2개를 살 수 있는 양 (전부는 안 됨).
- 업그레이드는 "시작이 편해지는 것"(시작 골드, 후보 질)이지 "전투가 세지는 것"이 아니어야 2회차 난이도가 유지된다.
- 도전 규칙 보상은 기본 보상의 30~50% 추가.
