# ibookk 백엔드 로직 전수 감사 — 버그 리포트 (심각도별)

대상: ibookk-os 프로덕션 DB의 함수·트리거·제약·RLS 전체(= 앱의 실제 백엔드 코드베이스).
방식: 각 로직을 프로덕션에서 실제 실행/공격해 재현 → 수정 → 재검증 → 테스트 데이터 완전 삭제.
(앱의 React/API 라우트 소스는 BIG 저장소라 이 세션 접근 밖 — DB 계층만 대상.)

## ✅ 감사했으나 문제 없던 것 (방어 장치 정상)
- 불균형 분개 거부, 게시분개 라인 불변, 게시분개 삭제 차단 → 정확히 작동
- 결제 금액 양수 강제, 분개 라인 단면·비음수 강제
- 중복 방지 unique: 계정코드/owner_basis/멤버/거래(Plaid+hash)/은행계좌
- 역분개 원자성, 연말마감 P&L 제로화·이익잉여금, 재분개
- RLS 테넌트 격리(테이블·시산표뷰·조직 가로채기 차단)
- Form 7203 검산, 1120-S 준비도, MTD 통계, 원장 잔액/월별 P&L 계산

## ✅ DB 계층 감사 완결 (2026-07-13)
프로덕션 DB의 함수·트리거·제약·RLS를 전수 감사했습니다.
- **트리거 배선 전수 확인**: updated_at 32개 + 원장 가드(균형/마감/불변/게시)·결제 재계산 전부 정상 연결
- **SECURITY DEFINER 함수**: 전부 search_path 고정 + anon 실행 차단 (미고정/노출 0)
- **RLS 커버리지**: 모든 테이블 RLS 활성 + 정책 보유 (정책 0개/RLS 꺼짐 테이블 0)
- **금액 제약**: 원장·결제·자산·급여·정보신고 전 영역 비음수/정합성 가드 완비
- **감가상각·compliance·급여·자산처분** 데이터 무결성 제약 추가 완료
